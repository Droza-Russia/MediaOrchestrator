# Архитектура MediaOrchestrator

## Обзор

MediaOrchestrator — это enterprise-grade .NET SDK для медиаобработки, адаптивного выполнения, аналитики и наблюдаемости. SDK предоставляет единый интерфейс для конвертации медиа, зондирования и интеллектуального выбора стратегии на основе исторических данных выполнения.

---

## 1. Программная архитектура

### 1.1 Структура слоёв

```
┌─────────────────────────────────────────────────────────────────┐
│                    Публичный API слой                           │
│  MediaOrchestrator (Facade), MediaSource, MediaDestination     │
├─────────────────────────────────────────────────────────────────┤
│                 Высокоуровневые операции                         │
│  Conversion Snippets, Analytics, Reporting                      │
├─────────────────────────────────────────────────────────────────┤
│                    Основной слой обработки                       │
│  Conversion, Probe, MediaInfo, Streams                          │
├─────────────────────────────────────────────────────────────────┤
│                    Инфраструктурный слой                         │
│  Кэширование, Хранилища, I/O, Определение HW                   │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 Ключевые модули

| Модуль | Ответственность |
|--------|-----------------|
| **MediaOrchestrator** | Статический фасад, точка входа, конфигурация |
| **Conversion** | Выполнение FFmpeg, прогресс, сниппеты |
| **Probe** | Извлечение и кэширование MediaInfo |
| **Analytics** | Адаптивное обучение, отчёты, хранилище |
| **Media** | Универсальный I/O (файлы, потоки, байты) |
| **HardwareAcceleration** | Автоопределение, профили |
| **Streams** | Модели видео/аудио/субтитров |
| **Localization** | Сообщения об ошибках, мультиязычность |

### 1.3 Целевая платформа

- **netstandard2.0** с совместимостью C# 7.3
- Поддержка .NET Framework, .NET Core, .NET 5+

### 2.2 Ключевые модели данных

#### MediaProcessingPlan
- **Scenario**: BrowserPlayback, AiTranscription, Custom
- **Strategy**: Remux, NormalizeAudio, FullTranscode
- **Constraints**: максимальная длительность, размер, кодеки, аппаратное ускорение
- **Reason**: Объяснение решения

#### MediaAnalysisRecord
- **ProbeSnapshot**: контейнер, кодеки, длительность, разрешение
- **ProcessingPlan**: выбранная стратегия и параметры
- **ExecutionSamples**: недавние выполнения с успехом/неудачей/скоростью
- **AggregatedStats**: статистика по стратегиям: успешность, средняя скорость

#### ExecutionResourceMetrics
- Пиковый рабочий набор (память)
- Загрузка CPU
- Загрузка аппаратного ускорителя (GPU)
- Логические ядра

## 3. Бизнес-архитектура

### 3.1 Домен проблем

| Проблема | Решение |
|----------|---------|
| Медиаобработка требует экспертизы FFmpeg | Высокоуровневые сниппеты (ToMp4, ExtractAudio и т.д.) |
| Неэффективное повторное зондирование | In-memory LRU-кэш с дедупликацией |
| Неоптимальный выбор стратегии | Адаптивная аналитика с историческим обучением |
| Нет видимости процесса обработки | Публичные аналитические отчёты с разбивкой |
| Хрупкие операции | Паттерн Circuit Breaker |
| Непредсказуемые таймауты | Адаптивный CancellationToken на основе истории |
| Надёжность I/O | Атомарные записи, логика повтора, буферный пул |

### 3.2 Варианты использования

#### Основные варианты
1. **Конвертация медиа** — конвертация между форматами (MP4, WebM, MKV и др.)
2. **Извлечение аудио** — извлечение аудиодорожек для AI-транскрибации
3. **Транскодирование** — транскодирование с аппаратным ускорением
4. **Зондирование медиа** — получение метаданных без полной конвертации
5. **Адаптивная обработка** — позволить системе выбрать оптимальную стратегию

#### Варианты использования аналитики
1. **Исторический анализ** — запрос истории обработки по сценарию/стратегии
2. **Мониторинг производительности** — отслеживание успешности, времени обработки
3. **Мониторинг ресурсов** — CPU, память, использование GPU
4. **Анализ отказов** — разбивка по коду ошибки, типу отказа

### 3.3 Ценностное предложение

| Заинтересованная сторона | Ценность |
|--------------------------|----------|
| **Разработчики** | Простой API, не требуется знание FFmpeg |
| **DevOps** | Проверки здоровья, метрики, предсказуемое поведение |
| **Операции** | Аналитические дашборды, отчёты об отказах |
| **Бизнес** | Адаптивная оптимизация, снижение затрат |

### 3.4 Модель ошибок

Все исключения наследуются от `MediaOrchestratorException` с:
- **ErrorCode**: читаемый идентификатор (например, `AudioStreamNotFound`)
- **ErrorCodeId**: стабильный ID (например, `MOR-IN-3007`)

Домены ошибок:
- `MOR-IN-*` — Ошибки ввода
- `MOR-CV-*` — Ошибки конвертации
- `MOR-IO-*` — Ошибки ввода/вывода
- `MOR-HW-*` — Ошибки аппаратного ускорения
- `MOR-TL-*` — Ошибки toolchain
- `MOR-HD-*` — Ошибки загрузки хостированного видео
- `MOR-OP-*` — Ошибки жизненного цикла операции
- `MOR-GN-*` — Общие ошибки

## 4. Паттерны устойчивости

### 4.1 Circuit Breaker
```
        ┌─────────┐
        │ Closed  │ ───► Нормальная работа, подсчёт ошибок
        └────┬────┘
             │ превышен порог ошибок
             ▼
        ┌─────────┐
        │  Open   │ ───► Быстрый отказ, нет операций
        └────┬────┘
             │ таймаут истёк
             ▼
    ┌────────┴────────┐
    │   Half-Open    │ ───► Тест восстановления
    └────────┬────────┘
             │ успех
             ▼
        ┌─────────┐
        │ Closed  │
        └─────────┘
```

### 4.2 Адаптивный таймаут
- Использует исторические данные о длительности операции
- Вычисляет взвешенное среднее с коэффициентом безопасности (2.0x)
- Корректирует при низкой успешности (<80%)
- Границы: мин 30с, макс 30мин

### 4.3 Логика повтора
- 3 попытки для файловых операций
- 100мс задержка между попытками
- Атомарные записи с временным файлом + переименование

## 5. Оптимизации производительности

| Оптимизация | Описание |
|-------------|----------|
| **LRU Кэш** | Кэширование результатов зондирования с TTL |
| **Дедупликация параллельного зондирования** | SemaphoreSlim предотвращает повторные вызовы ffprobe |
| **Буферный пул** | Переиспользуемые байтовые буферы для I/O |
| **Пул строковых интернов** | Оптимизация повторяющихся строк кодеков/форматов |
| **Атомарные операции** | Блокировочный сбор телеметрии |
| **Кэш определения HW** | Один ffprobe-вызов для -hwaccels |
| **Отложенная запись** | Асинхронная персистентность, не блокирующая |

## 7. Сводка публичного API

### Инициализация
```csharp
MediaOrchestrator.SetExecutablesPath("/path/to/ffmpeg", tryDetectHardwareAcceleration: true);
```

### Универсальный I/O
```csharp
MediaSource.FromFile(path);
MediaSource.FromStream(stream, ".mp4");
MediaSource.FromBytes(bytes, ".mp4");

MediaDestination.ToFile(path);
MediaDestination.ToStream(stream);
MediaDestination.ToBytes(".mp4");

MediaDirectoryDestination.ToDirectory(path);
MediaDirectoryDestination.ToMemory();
```

### Конвертация
```csharp
var conversion = await MediaOrchestrator.Conversions.FromSnippet.ToMp4(input, output);
await conversion.Start();
```

### Аналитика
```csharp
var report = await MediaOrchestrator.GetMediaAnalyticsReportAsync(query);
var metrics = MediaOrchestratorMetrics.Instance.GetSnapshot();
var health = MediaOrchestratorHealth.Check();
```

### Сброс
```csharp
await MediaOrchestrator.FlushMediaAnalysisStoreAsync();
```
